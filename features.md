# ポモドーロタイマー 実装機能一覧

このドキュメントは、`architecture.md` に基づき、ポモドーロタイマーアプリで実装が必要な機能を整理したものです。

## 1. MVPで必須の機能

### 1.1 タイマー基本機能
- 作業セッション開始（デフォルト25分）
- 一時停止 / 再開
- リセット（初期状態へ戻す）
- 残り時間表示（`mm:ss`）
- セッション完了検知（0秒到達）

### 1.2 セッション遷移機能
- 作業完了後に休憩へ遷移
- 通常は短休憩、一定回数後に長休憩へ遷移
- 現在状態の表示（作業中 / 休憩中）

### 1.3 進捗表示機能（UIモック対応）
- 円形プログレスの進捗更新
- 今日の完了ポモドーロ数表示
- 今日の集中時間（合計）表示

### 1.4 データ永続化機能
- セッション完了記録保存（`type`, `duration`, `completed_at`）
- 今日の統計取得API
- 設定値取得API

### 1.5 設定機能（最低限）
- 作業時間設定
- 短休憩時間設定
- 長休憩時間設定
- 長休憩までのインターバル設定（例: 4回ごと）

### 1.6 バリデーション / エラーハンドリング
- API入力チェック（負値、不正type、日時形式不正）
- フロント側エラー表示（保存失敗時の通知など）

---

## 2. 品質担保に必須の機能（テスト容易性）

### 2.1 状態機械ロジック（純粋関数）
- `START / PAUSE / RESUME / RESET / COMPLETE_SESSION` の遷移実装
- 不正操作ガード（例: `idle` で `RESUME` 不可）

### 2.2 時刻抽象化（Clock）
- `SystemClock`（本番）
- `FixedClock`（テスト）
- 日付境界テストを可能にする

### 2.3 Repository分離
- `SessionRepository`
- `SettingsRepository`
- Fake Repository によるユニットテスト

### 2.4 App Factory化
- `create_app(config)` によるアプリ初期化
- テスト設定（`TESTING=True`、テストDB）の切り替え

---

## 3. テストとして実装すべき項目

- 作業完了 → 短休憩遷移
- 4回目作業完了 → 長休憩遷移
- pause/resume時の残り時間整合性
- resetで初期化
- 今日の統計が当日分のみ集計される
- 日付境界（23:59→00:00）の集計が正しい
- API不正入力で `400` を返す

---

## 4. 準MVP（余力があれば早めに実装）

- ページ再読み込み時のセッション復元（`localStorage`）
- セッション完了時の通知音 / ブラウザ通知
- アクセシビリティ改善（キーボード操作、コントラスト）

---

## 5. 将来拡張（フェーズ2以降）

- ユーザー認証・アカウント管理
- 複数デバイス同期
- 週次/月次レポート
- タスク紐付け（何に集中したか）
- テーマ切り替え（ライト/ダーク）

---

## 6. 実装優先順（推奨）

1. タイマー基本機能 + 円形プログレス
2. セッション遷移 + 状態表示
3. セッション保存 + 今日の進捗API
4. 設定API + 設定UI
5. バリデーション + エラーハンドリング
6. 状態機械ユニットテスト
7. API統合テスト
8. 準MVP機能（復元・通知・アクセシビリティ）
